## Description

Processes chat messages using an LLM model with function calling capabilities, featuring two steps: chat for message processing and handle_tool_result for managing tool execution results.

Notes:
- Supports optional thread history via `use_thread_history` to include prior conversation turns.
- Exposes two steps:
  - `chat(message)`: main entry that processes user input and may emit tool calls.
  - `handle_tool_result(tool_call_id, tool_result)`: feeds tool results back into the conversation when needed.

## Example(s)
### Example 1: Simple chat turn
- Set `use_thread_history=false`.
- Call `chat(message="Summarize the document")`.
- The block generates a response; if tool calls are required, it requests them via function calling.

### Example 2: Handling a tool result
- After `chat` emits a tool call, execute the tool externally and pass the result back:
- Call `handle_tool_result(tool_call_id, tool_result)`.
- Then call `chat(message="continue")` to allow the LLM to integrate tool results.

## Error Handling
 - Tool or model errors propagate as block errors with descriptive messages.
 - If a required tool result is missing or malformed, the block will raise an error during `handle_tool_result`.

## FAQ
???+ question "How do I enable multi-turn context?"

    Set `use_thread_history=true`. The block will incorporate past messages into the prompt context.

???+ question "How are tool calls represented?"

    The LLM uses structured function calling. The block exposes `handle_tool_result` to receive results and continue the conversation.
